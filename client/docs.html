Cimexis.io Docs

General

Cimexis.io is a multiplayer shooter that supports the easy creation of bots to play competitively along side players. The documentation here will cover the basics of the game as well as how to put together a simple bot of your own.

Stats

Stats affect the strengths and weaknesses of a player. Each player is given 400 points which can be distributed in any way with up to 100 points allowed per stat.

  • health
    The maximum amount of health a player has.
  • regen
    How fast health regenerates over time.
  • speed
    The speed a player is able to move.
  • size
    The physical size of a player.
  • sight
    How far a player can see.
  • meleeDamage
    How much damage a player inflicts when touching another player. (cactus damage)
  • reload
    How fast a player can reload between shots.
  • damage
    How much damage bullets shot by the player will do. This does not affect meleeDamage.
  • bulletSpeed
    How fast a player's bullets move.
  • bulletSize
    The physical size of a player's bullets
  • knockback
    How hard other players will be knocked back when hit by the player's bullet.
  • kickback
    How much kickback the player will feel when shooting.
  • accuracy
    How accurate the player's shots will be to where they are aiming.

Bots

The following section will cover the basics of creating a bot to play the game.

Setup

Before you can get started you will need to download the bot kit. This kit contains some boilerplate code that will help you get started. Once you have downloaded the kit locate the bot.js file, this is the only file you need to change.

You can open and run this kit using an IDE such as Visual Studio code. You will also need to have Node.js installed. Once this is done there is a start script that will allow you to run the bot. While the bot is running you can at any point go to localhost:5000 to watch it in real time.

Basic

Before spawning the constructor will be called. Within the constructor you will be allowed to set your username, your color, your stats and your weapon.

Every 10th of a second the control method will be called. In this control method you are able to set how the bot will act on the next frame. This is done using the Functions described below.

Functions

These are built-in functions which you can use while building a bot. These functions are listed below.

  • getDist(tar) returns number

    Gets the distance between the player and the target vector or entity

    Parameters

    • tar Vector or Entity - The target to check the distance for

  • getState() returns State

    Gets the state of every entity that is visible to the player

  • getPlayerList(includePlayer) returns Player[]

    Gets a list of visible players

    Parameters

    • (optional) includePlayer boolean - If the player should be included in the list, by default false

  • getBulletList() returns Bullet[]

    Gets a list of visible bullets

  • getBulletListEnemy() returns Bullet[]

    Gets a list of visible bullets that can damage the player

  • getBulletListFriendly() returns Bullet[]

    Gets a list of visible bullets shot by the player

  • getClosest(tarList) returns Entity

    Get the closest entity to the player in a list

    Parameters

    • tarList Entity[] - The targetList to search for the closest in

  • getFurthest(tarList) returns Entity

    Get the furthest entity from the player in a list

    Parameters

    • tarList Entity[] - The targetList to search for the furthest in

  • getPlayer() returns Player

    Gets the player

  • runAt(tar)

    Run towards a target Vector or Entity

    Parameters

    • tar Vector or Entity - The target to run at

  • runFrom(tar)

    Run away from a target Vector or Entity

    Parameters

    • tar Vector or Entity - The target to run from

  • strafeAt(tar,direction)

    Run sideways compared to a target Vector or Entity

    Parameters

    • tar Vector or Entity - The target to strafe at
    • (optional) direction boolean - Specifies the direction to strafe in

  • shootAt(tar)

    Shoot towards a target Vector or Entity

    Parameters

    • tar Vector or Entity - The target to shoot at

  • shootFrom(tar)

    Shoot away from target Vector or Entity

    Parameters

    • tar Vector or Entity - The target to shoot away from

  • getMapSize() returns Vector

    Returns a Vector for how large the game map is.

  • stabilizeMovement(amount)

    This will exagerate movement requests to the server. This is useful for laggy connections or exagerating percise movements.

    Parameters

    • (optional) amount number - The amount the movement will be exagerated by, 5 by default
