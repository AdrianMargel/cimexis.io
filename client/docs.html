Cimexis.io Docs

General

Cimexis.io is a multiplayer shooter that supports the easy creation of bots to play competitively along side players. The documentation here will cover the basics of the game as well as how to put together a simple bot of your own.

Stats

Stats affect the strengths and weaknesses of a player. Each player is given 400 points which can be distributed in any way with up to 100 points allowed per stat.

  • health
    The maximum amount of health a player has.
  • regen
    How fast health regenerates over time.
  • speed
    The speed a player is able to move.
  • size
    The physical size of a player.
  • sight
    How far a player can see.
  • meleeDamage
    How much damage a player inflicts when touching another player. (cactus damage)
  • reload
    How fast a player can reload between shots.
  • damage
    How much damage bullets shot by the player will do. This does not affect meleeDamage.
  • bulletSpeed
    How fast a player's bullets move.
  • bulletSize
    The physical size of a player's bullets
  • knockback
    How hard other players will be knocked back when hit by the player's bullet.
  • kickback
    How much kickback the player will feel when shooting.
  • accuracy
    How accurate the player's shots will be to where they are aiming.

Bots

The following section will cover the basics of creating a bot to play the game.

Setup

Before you can get started you will need to download the bot kit. This kit contains some boilerplate code that will help you get started. You will also need to have node.js installed.

Basic

Before spawning the constructor will be called. Within the constructor you will be allowed to set your username, your color, your stats and your weapon.

Every 1/10th of a second the control method will be called. In this control method you are able to set how the bot will act.

Methods

There are a number of built-in functions which you can use while building a bot. These functions are listed below.

  • function getDist(tar) returns number

    Gets the distance between the player and the target vector or entity

    Parameters

    tar Vector or Entity

  • function getState() returns State

    Gets the state of every entity that is visible to the player

  • function getPlayerList(includePlayer) returns Player[]

    Gets a list of visible players

    Parameters

    (optional) includePlayer boolean

  • function getBulletList() returns Bullet[]

    Gets a list of visible bullets

  • function getBulletListEnemy() returns Bullet[]

    Gets a list of visible bullets that can damage the player

  • function getBulletListFriendly() returns Bullet[]

    Gets a list of visible bullets shot by the player

  • function getClosest(tarList) returns Entity

    Get the closest entity to the player in a list

    Parameters

    tarList Entity[]

  • function getFurthest(tarList) returns Entity

    Get the furthest entity from the player in a list

    Parameters

    tarList Entity[]

  • function getPlayer() returns Player

    Gets the player

  • function runAt(tar)

    Run towards a target Vector or Entity

    Parameters

    tar Vector or Entity

  • function runFrom(tar)

    Run away from a target Vector or Entity

    Parameters

    tar Vector or Entity

  • function strafeAt(tar,direction)

    Run sideways compared to a target Vector or Entity

    Parameters

    tar Vector or Entity

    (optional) direction boolean

  • function shootAt(tar)

    Shoot towards a target Vector or Entity

    Parameters

    tar Vector or Entity

  • function shootFrom(tar)

    Shoot away from target Vector or Entity

    Parameters

    tar Vector or Entity

  • function getMapSize() returns Vector

    Returns a Vector for how large the game map is.

  • function stabilizeMovement(amount)

    This will exagerate movement requests to the server. This is useful for laggy connections or exagerating percise movements.

    Parameters

    amount number
